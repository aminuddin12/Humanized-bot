# SSR & Dynamic Template Configuration Report

Laporan migrasi arsitektur Nuxt 4 dengan konfigurasi Server-Side Rendering (SSR) tingkat Enterprise dan dinamisasi komponen.

## Konfigurasi Utama

### Core SSR (nuxt.config.ts)
- **SSR Enabled**: `ssr: true` diaktifkan secara eksplisit.
- **Compatibility**: Menggunakan `future.compatibilityVersion: 4` untuk standar Nuxt terbaru.
- **Theming**: Konfigurasi `@nuxt/ui` untuk menggunakan palette `emerald` dan `cool` gray sebagai tema primer.

### App Shell & Entry Point
- **app.vue**: Bertindak sebagai pembungkus utama dengan `<NuxtLayout>` dan `<NuxtPage />`. Implementasi `useHead` untuk title template dinamis dan metadata SEO.
- **error.vue**: Halaman penanganan error (404/500) yang elegan dengan catch parameter `error` dan integrasi layout `full`.

## Arsitektur Layout Dinamis

1. **default.vue**: Layout standar yang mengintegrasikan `PublicNavbar` dan `PublicFooter`. Menu navigasi dikontrol melalui props `menuItems` untuk fleksibilitas CMS di masa depan.
2. **full.vue**: Layout khusus edge-to-edge untuk halaman landing atau error yang membutuhkan kontrol penuh atas viewport.

## Dinamisasi Komponen (Variant-Driven)

Semua komponen telah direfaktorisasi untuk menghilangkan hardcoded string:

- **PublicNavbar**: Menerima `menuItems` dan `logoLabel`. Menggunakan `NuxtLink` untuk routing internal.
- **HeroSection**: Menerima headline, highlightedText, subline, dan konfigurasi CTA secara dinamis.
- **FeatureGrid**: Menerima `bodyZones`, `title`, dan `description`. State aktif dikelola secara internal namun didorong oleh props eksternal.
- **DoctorList**: Menerima data array `doctors` yang disalurkan dari state halaman.

## Simulasi Data (index.vue)

Halaman utama bertindak sebagai "Data Orchestrator" dengan state reaktif:
```javascript
const pageData = reactive({
  hero: { ... },
  triage: { ... },
  doctors: { ... }
})
```
Data ini disalurkan ke komponen anak menggunakan `v-bind`, mensimulasikan integrasi API/CMS di masa mendatang dengan tetap menjaga kemurnian komponen UI.

## Verifikasi Teknis
- **Server Pre-rendering**: Berhasil memuat state awal di sisi server.
- **SEO Ready**: Tag title dan meta deskripsi terisi secara otomatis.
- **Clean Code**: Tidak ada komentar di semua file `.vue` dan `.ts`.
